---
name: grilling-codex
description: Codex-optimized grilling loop for stress-testing a plan or design. Use when the user wants a relentless design interview in Codex, with related questions grouped via `request_user_input`.
---

Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one. For each question, provide your recommended answer.

Ask 1 to 3 related questions at a time using `request_user_input`, waiting for feedback on each batch before continuing. Group questions only when they belong to the same decision point or unlock the same next branch.

If a question can be answered by exploring the codebase, explore the codebase instead.
